ACTIONS THAT A PLAYER PERFORMS ON OBJECTS (Object‑Acting Verbs):
attack // Use an item to harm or damage another object or entity
cut // Divide or open something using a sharp tool
ignite // Set something on fire or cause it to burn
inspect // get the properties of object, location
hook // Catch or pull something using a hook
shoot // Fire a projectile from a weapon
unlock // Open something that is locked using a key or similar item
drink // Consume a liquid
store_items // Place items into a container for storage
verify_identity // Confirm the identity of a character using a document
warm // Provide warmth, typically by using a blanket or similar item
wear // Equip clothing (gloves, mask, single_shoe)
sit // Sit on an object (plastic_chair)
give // give some item to another character
go // enter some location / sub_location
pickup // take any onject from the container
punch // punch the object / character
signal // signal
put. // place the item somewhere

the key algorithmetic logic: 

if Action if performed over some object: 

1)Check if item functionality allows it. 

for example if the clothing functionality is wear => 
and the user prompting is to wear the item, then the action is going to be performed
+ the effect is going to be applied. 

so how it works is:



User action 
    ↓
Does functionality exist?
    ├─ No → Do nothing
    └─ Yes
         ↓
Is it tool-based functionality?
    ├─ Yes
    │     ↓
    │  Is there an on-active subject? (target object)
    │         ├─ No → Do nothing
    │         └─ Yes → Do action on OBJECT Y using TOOL X
    │
    └─ No
         ↓
     Is object required?
           ├─ No → Do action (self-action)
           └─ Yes
                 ↓
          Is object present?
                 ├─ No → Do nothing
                 └─ Yes → Do action on object
